Babel & 번들링 개념 정리

Posted on 2025년 11월 16일
#React #Bundling #Babel #Webpack #Vite

# Babel이란?

최신 JavaScript(ES6+, JSX 등)를 브라우저가 이해할 수 있는 구버전 JavaScript로 변환(transpile)해주는 도구다.

브라우저는 순수한 JavaScript(ES5 스타일)만 이해할 수 있기 때문에, 중간에서 Babel이 코드 변환을 담당한다.

# Babel이 하는 일

//  개발자가 작성한 코드 (JSX + ES6+)
const Hello = () => <h1>안녕하세요</h1>;
const greet = (name) => `Hello, ${name}!`;

//  Babel이 변환한 코드 (브라우저가 이해할 수 있는 ES5)
var Hello = function () {
  return React.createElement("h1", null, "안녕하세요");
};

var greet = function (name) {
  return "Hello, " + name + "!";
};
변환 전 변환 후
JSX (<h1>) React.createElement() 호출
화살표 함수 (=>) 일반 함수 (function)
템플릿 리터럴 (` `) 문자열 연결 (+)
const, let var

# ✅ 번들링(Bundling)이란?

여러 개의 파일(JS, CSS, 이미지 등)을 브라우저가 읽을 수 있도록 하나(또는 몇 개)로 묶는 과정이다. 이 작업을 자동으로 해주는 도구를 번들러(Bundler) 라고 한다.

# 번들러가 필요한 이유

1. 프로젝트는 파일이 수백, 수천 개가 될 수 있다 React로 개발하면 컴포넌트, 스타일, 이미지 등 수많은 파일이 생긴다. 브라우저가 이 파일들을 각각 따로 요청하면 네트워크 비용이 커져 느려진다.

2. 최적화 작업이 필요하다

최적화 설명
코드 압축 (Minify) 공백, 주석 제거로 파일 크기 축소
트리 쉐이킹 (Tree-shaking) 사용하지 않는 코드 제거
번들 통합 여러 파일을 하나로 합쳐 요청 수 감소
코드 스플리팅 필요한 코드만 나눠서 로드 (Lazy Loading)

# ✅ 번들링 과정

[ 여러 개의 파일 (JS, CSS, 이미지) ]
               ↓
  [ Babel — JSX 및 최신 문법 변환 ]
               ↓
[ 번들러 — 파일을 하나로 묶고 최적화 ]
               ↓
   [ dist 폴더에 최종 결과물 생성 ]
               ↓
    [ 브라우저는 dist 폴더만 읽음 ]

# ✅ 주요 번들러 비교

번들러 특징
Webpack 가장 오래되고 유명. 설정이 많지만 강력하다. 레거시/대규모 프로젝트에서 주로 사용
Vite 요즘 대세. Webpack보다 훨씬 빠르고 간편하다. React/Vue/TS 등 지원
Parcel 설정 없이 바로 사용 가능. 빠른 시작용
Rollup 라이브러리 제작용 번들링에 최적화

현재 신규 React 프로젝트는 대부분 Vite를 기본으로 사용한다. Vite는 개발 중에는 번들링 없이 ESM 방식으로 바로 실행하고, 프로덕션 빌드 시에만 Rollup 기반으로 번들링한다.


핵심 요약:

  • Babel → 코드 변환 담당 (JSX, 최신 문법 → 브라우저가 이해하는 코드)
  • 번들러 → 파일 통합 및 최적화 담당
  • 실제 빌드 시 Babel과 번들러는 함께 동작한다
